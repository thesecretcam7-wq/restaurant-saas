📋 RESUMEN EJECUTIVO

Se implementó un sistema completo de:
- Período de prueba de 14 días para nuevos restaurantes
- Bloqueo automático de acceso después de período de prueba

- Webhooks para actualizar estado automáticamente
- Validación en middleware (no se puede saltear)
- Página de selección de planes (Basic, Pro, Premium)
- Panel de bloqueo cuando no hay suscripción activa

🎯 CARACTERÍSTICAS PRINCIPALES

1. PERÍODO DE PRUEBA AUTOMÁTICO
   - 14 días gratis para cada restaurante nuevo
   - created_at marca inicio
   - Validación automática en cada acceso

2. BLOQUEO DE ACCESO
   - Si no en trial Y no tiene suscripción activa → BLOQUEADO
   - No se puede saltear (en middleware)
   - Redirige a /subscription-blocked

3. TRES PLANES
   - Basic $29.99/mes (100 productos, 1000 órdenes/mes)
   - Pro $79.99/mes (500 productos, dominio personalizado)
   - Premium $199.99/mes (ilimitado, soporte 24/7, API)

4. STRIPE INTEGRADO
   - Checkout seguro
   - Cobro automático cada mes
   - Webhooks para actualizar estado

5. VALIDACIÓN AUTOMÁTICA
   - Layout valida SIEMPRE antes de cargar
   - Imposible saltear bloqueo
   - Transparente para el usuario

════════════════════════════════════════════════════════════════════════════

⚠️ LO IMPORTANTE QUE DEBES ENTENDER

1. DOS TIPOS DE DINERO:

   ÓRDENES (como antes):
   - stripe_account_id = Stripe Connected Account del restaurante
   - Dinero directo al restaurante
   - Plataforma NO toma comisión

   SUSCRIPCIONES (NUEVO):
   - stripe_customer_id = Tu cuenta de Stripe
   - Dinero para ti (modelo B2B)
   - Cada mes se cobra automáticamente

2. BLOQUEO ES DEFINITIVO:
   - No hay período de gracia
   - No hay email de advertencia
   - Después de 14 días, acceso cortado

3. WEBHOOKS SON CRÍTICOS:
   - Si no funcionan, pago se procesa pero status no actualiza
   - Usuario sigue bloqueado aunque pagó

📊 DIAGRAMA RÁPIDO DEL FLUJO

REGISTRO
   ↓
STATUS = 'trial' (created_at = ahora)
   ↓
14 DÍAS GRATIS (Acceso total)
   ├─ Si intenta acceder: ✅ OK
   └─ Banner: "X días left"
   ↓
VENCE EL DÍA 15
   ├─ Intenta acceder
   ├─ Layout valida
   └─ Redirige a /subscription-blocked ❌
   ↓
VE PLANES Y SELECCIONA (Ej: Pro $79.99/mes)
   ├─ POST /api/stripe/subscription
   ├─ Redirige a Stripe Checkout
   └─ Paga
   ↓
WEBHOOK RECIBE EVENTO
   ├─ customer.subscription.created
   ├─ UPDATE tenants: status='active', plan='pro'
   └─ Dinero llega a tu cuenta de Stripe
   ↓
PRÓXIMO ACCESO
   ├─ Layout valida: status='active'? Sí
   └─ ✅ Acceso permitido (sin trial)
   ↓
CADA MES
   ├─ Stripe intenta cobrar automáticamente
   ├─ Si éxito: continúa activo
   └─ Si falla: webhook → status='suspended' → bloqueado
